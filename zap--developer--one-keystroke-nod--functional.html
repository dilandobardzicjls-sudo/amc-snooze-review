Parallel agents, serial interruptions

Running several sessions means the pauses stack. Each one is trivial, a nod, but each nod used to cost a window switch, a scan for context, a retyped reply. Multiply by every session, every hour.

Zap makes the nod O(1). One keystroke per session, straight from the inbox, however many you’re running. The rows that need real answers still get you; the rest get Alt+Z, sent in under a second.

Run the sweep yourself

  1. Waiting sessions stack in the inbox: amber, flagged, one list.
  2. J/K moves between rows. The selected session opens right beside the list. No mouse, no window hunt.
  3. Alt+Z sends “keep going” to the open session. Its row clears, and the inbox drops you into the next one.

Attachments ride along; the same key works in your SMS and email threads; right-click the ⚡ button to set the exact phrase it sends (multiline is fine). The key itself is re-mappable in Settings → Keyboard.
